# ADR 0001: セキュリティと同意・プライバシー方針（PoC）

- Status: Proposed
- Date: 2025-12-11
- Owners: timtam PoC チーム

## 背景 / Context
本PoCは会議音声・文字起こし・介入内容を取り扱う。参加者の同意、PII（個人情報）や機微情報の扱い、データ保持期間、権限管理、監査が重要。ユーザー回答では「現時点では詳細は検討しない（TBD）」が前提だが、PoC段階での最小限の方針は必要。

## 決定 / Decision
- 同意・表示: 会議入室時に「AI介入・録音・ログ保存（PoC目的）」を明示（UI/チャットで自動告知）。TBD: 具体文面、拒否時の取扱い。
- データ保持: S3に暗号化保存（KMS）。保持期間は短期（例: 30日）を暫定値として設定。TBD: 正式期間、削除プロセス、自動ライフサイクル。
- アクセス制御: IAM最小権限。セッション単位のスコープドロールを採用。SecretsはSecrets Managerで管理。
- マスキング: PII簡易マスキング（電話/住所/メール等）を文字起こし整形時に適用可能な仕組みを用意。TBD: ルール詳細、例外。
- LLM利用: Amazon Bedrock経由を基本。Guardrailsを有効化し、特定カテゴリ（差別/ヘイト/機微）ブロック。TBD: ポリシー詳細。

（注）上記はPoCの仮決定。プロダクション前に法務・情報セキュリティ審査が必須。

## 影響 / Consequences
- 初期実装で告知・ログ・削除の基本機能が必要になる。
- ベンダーロックイン低減のため、データエクスポート（JSON/CSV）機能を前提に設計。
- Guardrailsにより一部出力が抑止され、介入頻度/内容に影響する可能性。

## 代替案 / Alternatives
- オンプレ/自前LLM推論による閉域運用（コスト/運用負担増）
- 外部DLP/PII検出サービスの導入（運用複雑性増）
